# System Flow

This document explains the runtime behavior of the HotReload engine.

## Startup Flow

When the tool starts:

1. CLI arguments are parsed
2. File watcher is initialized
3. Log dashboard server is started
4. Initial build is executed
5. Server process is started
6. File watching begins

## File Change Flow

When a developer modifies a file:

```
File modified
      │
      ▼
Watcher detects event
      │
      ▼
Debounce timer starts
      │
      ▼
Build command executes
      │
      ▼
If build succeeds:
      │
      ▼
Stop previous server
      │
      ▼
Start new server
```

## Logging Flow

Logs generated by the system are published to the LogHub.

```
Component logs message
       │
       ▼
LogHub publishes message
       │
       ├── Terminal output
       │
       └── Browser dashboard
```

This allows real-time visibility of the system state.
